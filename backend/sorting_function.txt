1. SYSTEM OVERVIEW
================================================================================

This system uses Google Gemini AI to match user profiles with projects based on
semantic understanding, not just keyword matching. It generates AI summaries of
both profiles and projects, creates embeddings (vector representations), and
uses cosine similarity to calculate match scores.

KEY FEATURES:
✓ AI-generated summaries for profiles and projects (2-3 sentences each)
✓ Vector embeddings created from AI summaries (768 dimensions)
✓ Cosine similarity matching (returns 0-1 score, displayed as percentage)
✓ Recommended projects on dashboard (top 3 matches)
✓ Explore page with "Sort by Match" toggle
✓ Random project order when NOT sorted by match
✓ Match percentage badges visible on both pages

6. DATA FLOW: MATCHING ALGORITHM (EXPLORE PAGE - SORT BY MATCH)
================================================================================

USER CLICKS "SORT BY MATCH" BUTTON
     │
     ├──► 1. Frontend: fetchProjects() called with sortByMatch=true
     │
     ├──► 2. API Call: GET /recommended-projects/{profile_id}?limit=50
     │
     ├──► 3. Backend: Calls Supabase RPC function
     │         Function: match_projects_to_profile(profile_uuid, 0.0, 50)
     │
     ├──► 4. Database RPC Function Executes:
     │         
     │         SELECT 
     │             proj.id,
     │             proj.title,
     │             ... (all project fields)
     │             1 - (proj.project_embedding <=> 
     │                  profile.bio_embedding) AS similarity
     │         FROM projects proj
     │         WHERE proj.project_embedding IS NOT NULL
     │             AND proj.status = 'open'
     │         ORDER BY similarity DESC
     │         LIMIT 50
     │
     │         The <=> operator calculates cosine DISTANCE (0 = identical)
     │         We use 1 - distance to get SIMILARITY (1 = perfect match)
     │
     ├──► 5. Backend returns:
     │         {
     │           "profile_id": "uuid",
     │           "recommended_projects": [
     │             {
     │               "id": "...",
     │               "title": "AI Study Assistant",
     │               "description": "...",
     │               "similarity": 0.873,  ← 87.3% match!
     │               ...
     │             },
     │             ...
     │           ]
     │         }
     │
     ├──► 6. Frontend: Maps similarity to similarity_score
     │         project.similarity_score = project.similarity
     │
     ├──► 7. Frontend: Displays projects with match badges
     │         "⭐ 87% Match" badge on each tile
     │
     └──► 8. User sees projects sorted by relevance!

  CREATE FUNCTION match_projects_to_profile(...)
  
  THIS IS THE MATCHING ALGORITHM!
  
  Returns columns: id, title, description, tags, owner_id, status, 
                   duration, availability_needed, project_image_url, 
                   roadmap, project_ai_summary, created_at, updated_at,
                   similarity
  
  SQL Logic:
    SELECT 
        proj.*,
        1 - (proj.project_embedding <=> 
             p.bio_embedding) AS similarity
    FROM projects proj
    WHERE proj.project_embedding IS NOT NULL
        AND proj.status = 'open'
        AND similarity >= match_threshold
    ORDER BY similarity DESC
    LIMIT match_limit
  
  The <=> operator: Cosine distance between vectors
  - Returns 0.0 when vectors are identical
  - Returns 2.0 when vectors are opposite
  
  We calculate: 1 - distance = similarity
  - similarity = 1.0 means 100% match (perfect alignment)
  - similarity = 0.0 means 0% match (no similarity)

10. HOW THE MATCHING ALGORITHM WORKS (TECHNICAL DEEP DIVE)
================================================================================

STEP 1: VECTOR EMBEDDINGS
─────────────────────────────────────────────────────────────────────────────
What are embeddings?
• Numerical representations of text
• Capture semantic meaning
• Similar concepts have similar vectors

Example:
  "Python developer interested in AI" → [0.23, -0.15, 0.08, ..., 0.42]
  "Machine learning engineer with Python" → [0.21, -0.14, 0.09, ..., 0.40]
  "Java backend architect" → [0.05, 0.22, -0.31, ..., 0.11]

Note: First two are similar (close vectors), third is different


STEP 2: COSINE SIMILARITY
─────────────────────────────────────────────────────────────────────────────
Measures angle between vectors:

                profile_embedding
                       /
                      / θ (angle)
                     /
                    /_________ project_embedding

Cosine Distance = 1 - cos(θ)
• When θ = 0° (same direction): distance = 0 (identical)
• When θ = 90° (perpendicular): distance = 1 (unrelated)
• When θ = 180° (opposite): distance = 2 (opposite meaning)

We calculate: Similarity = 1 - Distance
• 1.0 = perfect match (100%)
• 0.5 = moderate match (50%)
• 0.0 = no match (0%)

PostgreSQL operator: <=>
• Calculates cosine distance
• Optimized with HNSW index for fast lookups


STEP 3: THE MATCHING PROCESS
─────────────────────────────────────────────────────────────────────────────

Given:
• Profile embedding: P = [p1, p2, ..., p768]
• Project embedding: J = [j1, j2, ..., j768]

Calculate:
  distance = P <=> J  (using pgvector extension)
  similarity = 1 - distance

For all open projects:
  1. Get each project's embedding
  2. Calculate similarity with user's profile embedding
  3. Sort by similarity (descending)
  4. Return top N matches

Why it's smart:
• Understands "Python developer" matches "AI/ML project"
• Understands "beginner" matches "tutorial project"
• Understands "remote work" matches "async collaboration"
• Goes beyond just matching keywords!


STEP 4: THE AI SUMMARY ADVANTAGE
─────────────────────────────────────────────────────────────────────────────

Instead of embedding raw data:
  ❌ "Skills: [Python, JavaScript, React]. Major: Computer Science..."
  
We embed AI understanding:
  ✓ "This intermediate developer bridges modern web development and AI/ML,
     bringing Python, JavaScript, and React expertise. They excel in
     full-stack projects requiring both frontend polish and backend logic,
     making them ideal for cross-functional team collaborations."

The AI summary:
• Captures nuance and context
• Understands relationships between skills
• Infers project preferences
• Creates better embeddings
• Results in more accurate matching

15. SUMMARY
================================================================================

The matching system works in 3 stages:

STAGE 1: PREPARATION (One-time, during creation)
• Generate AI summary of profile/project
• Create vector embedding from summary
• Store both in database

STAGE 2: MATCHING (On-demand, when browsing)
• Use RPC function to calculate similarity
• Compare user's embedding with all project embeddings
• Return projects sorted by similarity score

STAGE 3: DISPLAY (Real-time, in UI)
• Dashboard: Show top 3 matches
• Explore page: Toggle between random and sorted
• Show match percentage badges
• Enable informed decision-making

This creates an intelligent matching system that understands context and 
semantics, not just keywords!
